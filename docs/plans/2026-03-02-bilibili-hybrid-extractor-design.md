# B 站多模态抽取工具 (Bilibili Hybrid Extractor)

## 项目背景
为突破 B 站反爬及验证码机制，构建一个允许 AI 代理安全稳定地读取视频内容的混合型提取工具。

## 架构方案
该工具采用“文本主导 + 视觉辅助”的混合架构：
1. **轻量级文本轨道 (Text Track)**：调用第三方库（如 `bilibili-api-python` 或类似的无头工具包）快速拉取视频元数据、字幕(CC)、热点高赞弹幕。用于 90% 的知识类、解说类视频的快速“阅读”。
2. **深度视觉轨道 (Visual Track)**：当视频高度依赖画面（例如风景、无解说演示、报错复现）时，被动触发。使用 `yt-dlp` 下载视频并利用 `ffmpeg` 提取关键帧（每 N 秒截图），提供画面序列给大模型审阅。

## 核心流程
1. 接收 B 站 URL 或 BV 号。
2. 基础元数据诊断：有没有字幕？有没有足够文本量（评论/弹幕）？
3. 自动降级返回：默认返回纯文本（视频信息概览 + 完整字幕 + 前 100 弹幕摘要）。
4. 在需要时，允许传入 `--visual` 标志，触发视频源流下载与抽帧组件。
